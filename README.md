# OpenFinder

OpenFinder is a discovery platform for open-source software.

Our goal is to make it easy to find high-quality open-source tools across domains like development, design, productivity, privacy, infrastructure, and AI — without the noise of unstructured lists, inconsistent quality, or the friction that often makes open-source tools difficult to discover, evaluate, and integrate into daily use.

---

## What makes OpenFinder different

OpenFinder is not a directory of everything.

It is built around:

- **Curation** — tools are selected for quality, usefulness, and relevance
- **Clear categorization** — tools are organized by purpose, not loosely grouped  
- **Structured tagging** — consistent, cross-cutting traits (see `/docs/taxonomy.md`)  
- **Readable descriptions** — concise, beginner-friendly summaries  
- **Guides & workflows** — structured resources that help users discover, compare, and get started with tools in real-world use cases  

The goal is to create a **high-signal index of the open-source ecosystem** — one that not only lists tools, but helps people of all experience levels actually use them.

---

## Launch Scope

The initial release includes:

- ~200 selected tools  
- standardized categories and subcategories  
- a controlled tagging system  
- searchable listings  
- initial guides, workflows, and tool alternatives for common use cases
